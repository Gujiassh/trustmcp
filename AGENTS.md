# TrustMCP Repo Notes

## Product Memory

- TrustMCP exists to become a narrow, trustworthy open-source MCP server security scanner for JavaScript and TypeScript repositories.
- The product goal is not to be a full security platform; it should help people audit risky MCP server capabilities before they run unknown code locally or wire it into CI.
- The star-growth thesis is strong README and discoverability, real GitHub Action and CI integration, honest scope, and low-friction first use.
- Near-term product direction is to keep improving real adoption surfaces such as CLI ergonomics, docs, examples, and release or install readiness before any broader promotion push.

## Repo Guidance

- Keep public docs aligned with shipped behavior.
- Use `docs/ssot/` for stable public-facing repo knowledge.
- Prefer narrow scope and clear claims over broader but fuzzier positioning.
